HTML

  • The head element contains information about the webpage.
  • The metadata in the head element informs the browser how to load the page.
  • Elements in the head element are not visible to the user.
  • The body element represents the visible content shown to the user.
  • Try to only use semantic elements in the body.

CSS

  • We can implement CSS inline, internally(in the head element), or externally. Generally we want to use an external style sheet as it's easier to maintain.
  • If using an external style sheet, be sure to link to it from your html file
  • A margin indicates how much space we want around the outside an element
  • A padding indicates how much space we want around the content inside an element
  • We can use a class attribute to apply a style across certain elements we choose.

Git

  • git status: checks what branch we are currently on
  • git checkout -b feature/branch-name: creates a new branch and switches to it
  • git add -A : stages all changes made to files to be ready to commit to the repository
  • git commit -m "add message text here" : commits to local repo with -m flagging the message accompanying the commit
  • git pull origin main : make sure local branch is up to date with main befure pushing changes
  • git push origin feature/branch-name : pushes the commit to a branch with name feature/branch-name

JavaScript

  • JavaScript allows us to add interactivity to a website
  • Primitive types include string, number, bigint, bool, array, symbol, null, and undefined.
  • A variable is a named container that allows us to store data in our code.
  • Define variables using keywords 'const' for constants or 'let' for all others. Avoid using var.
  • Control flow is the order in which a computer executes code in a script.
  • Conditionals (example: if statements) allow us to alter the control flow.
  • We can create a loop using the 'for' keyword to iterate as required.
  • Functions allow us to define a task and execute it when we wish.
